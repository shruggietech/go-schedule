# Changelog

The release history is maintained in the repository so that it sits alongside
the code it describes and is edited as part of each release. It follows the
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/) format, and the project
adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Each release section records not just what changed but, where a change touched a
pinned artifact, the **dated decision** behind it — the rationale a future reader
needs to understand why a load-bearing file looks the way it does.
